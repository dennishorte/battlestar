# Bare-string `choose()` Cleanup

Plan for eliminating `[choose] bare-string option` warnings emitted by
`BaseActionManager._warnOnBareStrings` (`common/lib/game/BaseActionManager.js`).
The warning fires whenever a `choose()` call receives bare strings instead of
structured `{title, id, kind?}` options. It is silenced under `NODE_ENV=test`
and `NODE_ENV=production`, so it only surfaces in dev play or when the test
suite is run with `NODE_ENV=development`.

This document covers **Agricola only**. Other games (Dune, Twilight, Ultimate,
etc.) will be addressed separately afterward, and the dev-time lock-in
(throw-on-warn) lands after all games are clean.

## Scope (Agricola)

Running the full Agricola suite with `NODE_ENV=development`:

- ~235,000 warning lines total
- 326 distinct prompt titles (~326 callsites)
- 310 `choose()` callsites across 239 source files (most in `res/cards/`)
- 277 `selection[0] === '...'` / `!==` matchers that need to flip to `.id`
- 17 parse-after-select sites (`.split(',')`, `choices.indexOf(...)`) that
  should also flip to id lookups

Distribution is top-heavy. The single "Choose an action" prompt produces 91%
of the warnings.

| Prompt | Warnings | Where |
|---|---|---|
| Choose an action | 214,782 | `phases/work.js:122` |
| Play an Occupation (free) | 9,877 | `cardManagement.js` |
| Choose an Improvement | 1,965 | improvement play flow |
| Choose what to build | 970 | build action |
| Need N more food (×4) | ~1,500 | feeding phase |
| Top 20 prompts | ~99% of total | core game flow |
| Long tail (~300 prompts) | ~1% | one-off card prompts |

## Target idiom

Use the existing `actions.option({ id, title, kind?, defId?, meta? })` helper.
Read results via `selection[0].id`.

```js
// Before
const choices = ['Build pasture', 'Skip']
const sel = game.actions.choose(player, choices, { title: '...' })
if (sel[0] === 'Skip') return

// After
const choices = [
  game.actions.option({ id: 'build', title: 'Build pasture' }),
  game.actions.option({ id: 'skip',  title: 'Skip' }),
]
const sel = game.actions.choose(player, choices, { title: '...' })
if (sel[0].id === 'skip') return
```

Rules:

- Convert **all** options at a callsite, not just the descriptive ones —
  mixing bare sentinels (`'Skip'`) with structured options yields mixed-shape
  responses.
- Action spaces → `{ id: actionId, title: action.name, kind: 'action-space',
  meta: { accumulated } }`.
- Cards (occupations / improvements) → prefer the existing `chooseCard()`
  helper, which already structures the response. Where it doesn't fit, use
  `actions.cardOption(card)`.
- Resources → `{ id: r, title: r, kind: 'resource' }`.
- Coordinate-derived prompts (Stable Manure, Clearing Spade, etc.) →
  `{ id: \`r${row}c${col}\`, title: \`(${row},${col}) ${crop} x${count}\` }`.
  Look up by id; do not parse the title back out.

Tests do not need changes. `TestCommon.choose` does title-based lookup and
automatically upgrades to `{title, id}` form for structured choices, so
existing `t.choose(game, 'Build pasture')` calls continue to work.

## Phases

### Phase 1 — high-leverage core prompts

Targets ~99% of warning volume by fixing a small number of central callsites.

- [ ] `phases/work.js` — "Choose an action" (single biggest win)
- [ ] `cardManagement.js` — occupation prompts (free / 1 food / 2 food);
      prefer migrating to `chooseCard()`
- [ ] Improvement play — "Choose an Improvement"; prefer `chooseCard()`
- [ ] `actions/building.js` — "Choose what to build", "Choose a major
      improvement", room/oven prompts
- [ ] `phases/feeding.js` — "Need N more food" family
- [ ] `actions/fencing.js` — "Continue fencing?", "Select spaces for pasture"
- [ ] Color selection — "Choose a player color"
- [ ] Anytime / scheduled handlers in `anytime.js`, `scheduled.js`

Expected warning count after Phase 1: a few hundred (only the card tail).

### Phase 2 — card-by-card cleanup

~200 card files in `res/cards/`. Done deck-by-deck:

- [ ] `minorA`
- [ ] `minorB`
- [ ] `minorC`
- [ ] `minorD`
- [ ] `minorE`
- [ ] `occupationA`
- [ ] `occupationB`
- [ ] `occupationC`
- [ ] `occupationD`

Each card is a ~5–15 line change. Stable, semantic ids (e.g. `'pay'`,
`'skip'`, `'convert-2'`) — not autogenerated slugs of the title.

### Phase 3 — verification

After each phase, re-run:

```bash
NODE_ENV=development npx jest --silent=false --testPathPattern='common/agricola' 2>&1 \
  | grep -E '^\s*\[choose\] bare-string' | wc -l
```

Target after Phase 2 completion: **0**.

Lock-in (CI gate that fails on the warning) is deferred until all games are
clean.
